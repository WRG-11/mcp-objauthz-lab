# Challenge S5 — The Unchecked Badge

**Difficulty:** Beginner–Intermediate
**Class:** CWE-863 — Role/token-type bypass
**Tool under test:** `note_admin_get`

## Setup

```bash
npm install
LAB_S5=vuln node src/server.js
```

## Your mission

The lab exposes a tool called `note_admin_get`, documented as "admin/support
use only." You are **Bob** (`bob-token`, org *Globex*) — an ordinary user,
not an admin.

Acme has a note: `n_acme_1`. You already know `note_get` denies you access
to it (correctly authorized).

1. Call `note_admin_get` with your ordinary `bob-token` and `n_acme_1`.
2. See whether the tool actually checks that you hold the admin role before
   answering — or whether the name alone was doing all the work.

## Rules

- Token: `bob-token` only. You are not an admin.
- You may NOT modify the server source.
- `LAB_S5=vuln` must remain set.

## Verification

The exploit works when the server returns `n_acme_1`'s content (not an
`AuthzError`) for `bob-token` — a token that holds no admin role.
