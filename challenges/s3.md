# Challenge S3 — The Leaky Batch

**Difficulty:** Intermediate  
**Class:** CWE-862 — Missing authorization (list→get asymmetry)  
**Tool under test:** `note_batch_get`

## Setup

```bash
npm install
LAB_S3=vuln node src/server.js
```

## Your mission

You are **Alice** (`alice-token`, org *Acme*).

Start by calling `note_list` with your token. Notice it returns only Acme's
notes — the list endpoint is correctly scoped.

Now, the note ids in this system are guessable: `n_acme_1`, `n_acme_2`,
`n_globex_1`, `n_globex_2`, `n_initech_1`, `n_initech_2`.

`note_get` will block you from reading foreign notes:

```json
{ "token": "alice-token", "id": "n_globex_1" }
→ AuthzError: cross-tenant access denied
```

Your challenge: use `note_batch_get` to read a **Globex** note and an
**Initech** note in the same request, while authenticated only as Alice.

## Constraints

- Token: `alice-token` only.
- `LAB_S3=vuln` must remain set.
- You may call `note_list` and `note_get` to confirm the correct baseline
  behavior, but the exploit uses `note_batch_get`.

## Verification

The exploit succeeds when `note_batch_get` returns notes with `"orgId":
"org_globex"` or `"org_initech"` alongside Alice's own notes.
