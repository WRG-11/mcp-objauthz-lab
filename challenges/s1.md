# Challenge S1 — The Single Outlier

**Difficulty:** Beginner  
**Class:** CWE-639 / CWE-862 — Object-level authorization  
**Tool under test:** One of the six core note tools

## Setup

```bash
npm install
LAB_MODE=vuln node src/server.js   # or: npm start
```

## Your mission

The lab is a multi-tenant note server. Three organizations share it: **Acme**
(Alice), **Globex** (Bob), **Initech** (Carol). Each user can only see and
touch their own organization's notes.

You are **Bob** (`bob-token`, org *Globex*). Acme has a note: `n_acme_1`.

1. Confirm that `note_get` denies you access to `n_acme_1`. The server does
   enforce authorization — somewhere.
2. Find the **one tool** that lets you touch Acme's note anyway, without any
   special privilege.
3. Use it. The exploit succeeds when you affect `n_acme_1` with Bob's token.

## Rules

- You may call any tool with any arguments.
- You may NOT modify the server source.
- `LAB_MODE=vuln` must remain set.

## Verification

The exploit works when the server returns a success response (not an
`AuthzError`) for an operation on `n_acme_1` using `bob-token`.
