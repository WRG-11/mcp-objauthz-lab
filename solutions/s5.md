# Solution S5 — The Unchecked Badge

## The exploit

Authenticate as Bob (`bob-token`, org *Globex*, an ordinary user) and call
`note_admin_get` with Acme's note id:

```json
{
  "name": "note_admin_get",
  "arguments": { "token": "bob-token", "id": "n_acme_1" }
}
```

In `LAB_S5=vuln` the server responds with the full note:

```json
{
  "id": "n_acme_1",
  "orgId": "org_acme",
  "ownerId": "u_alice",
  "title": "Acme acquisition memo",
  "body": "CONFIDENTIAL (Acme only): terms for the pending acquisition."
}
```

Bob — who holds no admin role and no Acme membership — just read Acme's
confidential note through a tool that was supposed to be admin-only.

## Why it works

`note_admin_get` in `src/tools.js` (vuln mode):

```js
guard(async ({ token, id }) => {
  const session = resolveSession(store, token);
  // requireAdminRole is MISSING here
  const note = store.getNote(id);
  if (!note) notFound(id);
  return ok(note);
}),
```

The tool's name and description say "admin/support use only" — but that is
documentation, not enforcement. The server correctly resolves *who Bob is*
(an ordinary Globex user) but never checks *what role Bob holds* before
answering. Any valid token reaches the cross-org lookup.

## The fix

`LAB_S5=fixed` adds the missing role check before resolving the note:

```js
if (modes.s5 === "fixed") requireAdminRole(session); // ← THE FIX
```

`requireAdminRole()` (in `src/auth.js`) throws `AuthzError` unless
`session.role === "admin"`.

## Two-way confirmation

- **Vuln + Bob (ordinary user)** → LEAKED ✓ (exploit reproduces)
- **Fixed + Bob (ordinary user)** → DENIED ✓ (fix blocks it)
- **Fixed + Dana (real admin, `dana-token`)** → ALLOWED ✓ (fix does not
  over-block legitimate admin use)

## Key takeaway

This is the "role/token-type bypass" pattern. A tool's name or description
promising a privilege tier ("admin", "service", "internal") is not a
security control by itself — the handler must actually read the
server-trusted role from the session and check it, the same way it checks
org membership. A badge that says "admin" on the box means nothing if
nobody checks it at the door.
